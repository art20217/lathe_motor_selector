# X / Z 軸伺服馬達選型 — 升級計劃

> **文件版本**：v1.0（規劃階段，尚未實作）
> **建立日期**：2026-07-07
> **來源**：解構外部 Python 分析工具（`servo.py` / `physics.py` / `motors.py`）的軸向選型模組
> **目標**：將 X/Z 軸伺服選型功能整合進本網頁工具，成為主軸選型（Phase 1–4）之後的延伸功能

---

## 1. 目標與範圍

### 1.1 目標

在現有「主軸馬達選型」流程之外，新增 **X / Z 軸進給伺服馬達選型**：

- 由 Phase 1 工況矩陣自動取得軸向切削分力（Z 軸 ← 進給分力 Ff、X 軸 ← 背分力 Fp）
- 支援兩種傳動方式：
  - **滾珠螺桿**（ballscrew）：X 軸常用，可加減速比
  - **雙電機＋排齒（齒條）減速機**（rack_dual）：大型長床身 Z 軸常用，master-slave 電氣預壓消隙
- 支援硬軌（box）/ 線軌（linear）摩擦模型，含起動靜摩擦倍率
- 五項檢核：加速峰值扭矩、切削持續扭矩、工作循環 RMS、快移轉速、慣量比
- 可編輯、可持久化的伺服馬達庫（沿用 Phase 2 馬達庫的模式）

### 1.2 不在範圍內

- 顫振穩定性、熱變位、動力刀塔 / C 軸、多刀同時切削（與來源工具相同的排除項）
- 螺桿臨界轉速（DN 值）、螺桿挫曲檢核（列為日後擴充，見 §8）
- 主軸側的機械損失模型（軸承摩擦、風阻）— 屬主軸升級題目，另案處理

## 2. 來源工具解構（移植依據）

以下公式與流程完整記錄自 Python 工具，作為移植時的對照基準。

### 2.1 軸需求計算（`servo.py: axis_analysis`）

**共通輸入**：移動質量 `mass` [kg]、滑軌型式、摩擦係數 μ（省略則依滑軌預設）、
軸傾角 θ [°]、快移速度 `rapid` [m/min]、加速時間 `ta` [s]、工作循環 duty
（預設 accel 10% / cut 50% / rapid 15% / idle 25%）、軸向切削力 `F_cut` [N]。

**摩擦力與重力分量**：

```
F_friction = μ·m·g·cos(θ)        [N]
F_gravity  = m·g·sin(θ)          [N]      (g = 9.81)
```

**滾珠螺桿**（效率 η_s = 0.90，螺帽預壓拖曳概估 T_preload = 2.0 N·m）：

```
conv     = lead / (2π·η_s·i)                          力 → 馬達端扭矩 [m]
J_screw  = ½·ρ_steel·π·r_screw⁴·L_screw               螺桿慣量 [kg·m²]
J_mass   = m·(lead / 2π)²                             直線質量等效慣量
J_load   = (J_screw + J_mass) / i²                    折算至馬達端
n_rapid  = rapid / lead × i                           快移馬達轉速 [rpm]
T_drag   = T_preload / i
```

（lead 單位 m/rev；i = 減速比 n_motor/n_screw ≥ 1；ρ_steel = 7850 kg/m³）

**雙電機排齒**（綜合效率 η_r = 0.90，減速機空轉拖曳 T_drag = 0.5 N·m/馬達端）：

```
r_p      = module·teeth / 2000                        小齒輪節圓半徑 [m]
conv     = r_p / (η_r·i·n_motors)                     力 → 每顆馬達扭矩
J_pinion = ½·ρ_steel·π·r_p⁴·(10·module/1000)          小齒輪慣量（齒寬概估 10×模數）
J_load   = (J_pinion + (m/n_motors)·r_p²) / i²        每顆馬達的折算慣量
n_rapid  = rapid / (2π·r_p) × i
bias_pct = 消隙預壓佔額定扭矩比例（預設 0.15）
```

**需求扭矩**（雙電機時為每顆馬達）：

```
T_friction = F_friction·conv + T_drag
T_gravity  = F_gravity·conv
T_cut      = F_cut·conv
T_cutting_total = T_cut + T_friction + T_gravity
```

### 2.2 五項檢核（`servo.py: evaluate_servo`）

```
bias_T  = bias_pct × T_rated                          消隙預壓扭矩
J_total = J_motor + J_load
ω_rapid = 2π·n_rapid / 60

T_accel = J_total·ω_rapid/t_a + stiction·T_friction + T_gravity
T_decel = |J_total·ω_rapid/t_a − T_friction − T_gravity|
T_rms   = √(d_accel·T_accel² + d_cut·T_cutting_total²
            + d_rapid·(T_friction+T_gravity)² + d_idle·T_gravity²) + bias_T

檢核（全部通過才列入推薦）:
  peak    : T_accel ≤ 0.8 × T_peak
  cutting : T_cutting_total + bias_T ≤ 0.8 × T_rated
  rms     : T_rms ≤ T_rated
  speed   : n_rapid ≤ n_max
  inertia : J_load / J_motor ≤ 10（> 5 顯示警告）
```

推薦清單依 T_rated 由小到大排序，取通過者。

### 2.3 滑軌特性（`physics.py: GUIDEWAY`）

| 型式 | μ（動摩擦） | 起動倍率（stiction） |
|---|---|---|
| 硬軌 box（貼塑滑動面） | 0.12 | ×1.8 |
| 線軌 linear（滾動導軌） | 0.01 | ×1.2 |

### 2.4 軸向切削分力來源（`materials.py` + `cutting.py`）

Python 工具在材料庫每筆資料附 `ff_ratio` / `fp_ratio`（進給分力、背分力相對主切削力
Fc 的經驗比例），計算 `Ff = ff_ratio·Fc`、`Fp = fp_ratio·Fc`，
再以 Z 軸 ← Ff、X 軸 ← Fp 帶入軸需求計算。

代表值（ISO 群）：P 鋼 0.40/0.30、M 不鏽鋼 0.45/0.35、K 鑄鐵 0.35/0.25、
N 鋁 0.35/0.25、S 耐熱合金 0.45–0.50/0.35–0.40、H 硬車 0.50/0.55。

### 2.5 伺服馬達庫（`motors.py: SERVO_MOTORS`）

Fanuc αiF/αiS-B 8 型 + Siemens 1FT7 5 型，欄位：`t_rated`（連續失速扭矩 N·m）、
`t_peak`（最大扭矩）、`rated_rpm`、`max_rpm`、`inertia`（kg·m²）。
**全部為型錄代表值，移植時須標記 `verified: false`。**

## 3. 資料模型設計

### 3.1 型別（`src/engine/types.ts` 擴充）

```ts
/** 進給軸伺服馬達 */
export interface ServoMotor {
  id: string
  brand: string
  model: string
  tRated: number        // 連續失速扭矩 [N·m]
  tPeak: number         // 最大扭矩 [N·m]
  ratedRpm: number
  maxRpm: number
  inertia: number       // 轉子慣量 [kg·m²]
  verified: boolean     // false = 型錄代表值未核對
  note: string
}

export type AxisDriveType = 'ballscrew' | 'rack_dual'
export type GuidewayType = 'box' | 'linear'

/** 單一進給軸配置 */
export interface AxisConfig {
  driveType: AxisDriveType
  movingMass: number            // [kg]
  guideway: GuidewayType
  frictionCoef: number | null   // null = 依滑軌預設
  stictionFactor: number | null // null = 依滑軌預設
  orientationDeg: number        // 軸傾角（斜床身 X 軸常見 30–60°）
  rapidSpeed: number            // [m/min]
  accelTime: number             // [s]
  duty: { accel: number; cut: number; rapid: number; idle: number }
  // ballscrew
  screwDia: number              // [mm]
  screwLead: number             // [mm/rev]
  screwLength: number           // [mm]
  gearRatio: number             // i = n_motor/n_screw ≥ 1
  // rack_dual
  pinionModule: number
  pinionTeeth: number
  rackGearRatio: number         // 排齒減速機減速比
  nMotors: number               // 預設 2
  biasPct: number               // 消隙預壓比例，預設 0.15
  /** 手動覆寫軸向切削力 [N]；null = 取 Phase 1 工況矩陣最大值 */
  cuttingForceOverride: number | null
}
```

`Material` 擴充兩個選填欄位（避免破壞既有自訂材料資料）：

```ts
export interface Material {
  // …既有欄位…
  ffRatio?: number   // Ff/Fc 進給分力比（Z 軸），未填以 ISO 群預設帶入
  fpRatio?: number   // Fp/Fc 背分力比（X 軸）
}
```

`DutyCase` 同樣加 `ffRatio?` / `fpRatio?`（建立工況時自材料複製，可覆寫），
`DutyResult` 加 `Ff: number | null`、`Fp: number | null`。

## 4. 計算引擎（新增 `src/engine/axis.ts`）

純函式模組，風格比照 `cutting.ts`（單一公式一個具名 export、JSDoc 註明單位與出處）：

| 函式 | 內容 | 對應來源 |
|---|---|---|
| `axisForces(mass, mu, theta)` | F_friction / F_gravity | §2.1 |
| `ballscrewDrive(cfg)` | conv、J_load、n_rapid、T_drag | §2.1 |
| `rackDualDrive(cfg)` | conv、J_load（每馬達）、n_rapid、T_drag | §2.1 |
| `axisRequirement(cfg, fCut)` | 彙整 T_cut/T_friction/T_gravity/ω | §2.1 |
| `evaluateServo(motor, req)` | 五項檢核 + T_accel/T_decel/T_rms | §2.2 |
| `recommendServos(motors, req)` | 通過者依 tRated 排序 | §2.2 |

**移植時的修正與強化**（相對 Python 原版）：

1. **軸向設計力取自工況矩陣**：`getAxisDesignForce(cases, results, 'x'|'z')` 取全工況
   `max(Fp)` / `max(Ff)`（Python 只算單一工作點，會漏掉最惡劣工況）。direct 模式工況
   無 Fc，排除並於 UI 註記。
2. **duty 比例驗證**：accel+cut+rapid+idle ≠ 1 時 UI 警告（Python 無驗證）。
3. **T_decel 納入檢核顯示**：Python 有算但未用；垂直/斜軸減速時重力助推，仍應對照
   T_peak 顯示。
4. **RMS 中 bias_T 的處理方式文件化**：Python 將消隙預壓加在 RMS 開根號之後
   （偏保守的近似）；移植維持同式以便 golden test 對齊，但在 JSDoc 註明此近似。
5. **單位一致性**：內部 SI（比照 `dynamics.ts` 慣例），UI 層負責 mm→m 換算。

## 8. 已知假設與風險

| 項目 | 說明 | 對策 |
|---|---|---|
| 螺帽預壓拖曳 2.0 N·m、減速機拖曳 0.5 N·m | Python 硬編碼概估值，實際依規格差異大 | 做成可覆寫欄位＋「假設值」Badge |
| 小齒輪齒寬 = 10×模數 | 慣量概估 | 同上，允許直接輸入 J_pinion 覆寫 |
| ff/fp 比例 | ISO 群經驗值，非量測 | 材料庫欄位可編輯＋報告標記 |
| 伺服馬達代表值 | t_peak 與過載時間相依，型錄曲線非單一值 | `verified: false` + 報告警語（比照主軸馬達庫） |
| 檢核門檻 0.8×T_peak / 0.8×T_rated / 慣量比 5/10 | 來源工具的工程慣例，無出處文件 | 常數集中定義並註明；日後可做成進階設定 |
| 未涵蓋：螺桿臨界轉速、DN 值、挫曲、齒條齒面強度 | 選型完整性缺口 | 列入日後擴充清單，報告註明未檢核項 |
| activePhase 編號變動 | 既有 localStorage 的 `activePhase: 5`（報告）語意改變 | persist migrate 時重映射 |
